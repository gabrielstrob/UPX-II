Instalar Chocolatey e Maven no Windows

Esse erro significa que o PowerShell não encontrou o comando choco, ou seja, o Chocolatey não está instalado (ou não está no PATH).

Veja abaixo como resolver e, também, como instalar o Maven sem o Chocolatey.

1. Verificar se o Chocolatey está instalado

No PowerShell, rode:

choco -v

Se aparecer o mesmo erro, é porque o Chocolatey realmente não está instalado.

2. Instalar o Chocolatey (Windows)

  1. Abra o PowerShell como Administrador.
  2. Verifique se a política de execução permite scripts: 
    Get-ExecutionPolicy
    Se retornar Restricted, execute: 
    Set-ExecutionPolicy Bypass -Scope Process -Force
  3. Execute o comando oficial de instalação do Chocolatey: 
    Set-ExecutionPolicy Bypass -Scope Process -Force; `
[System.Net.ServicePointManager]::SecurityProtocol = `
    [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; `
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
  4. Feche o PowerShell e abra novamente (como admin ou não, tanto faz).
  5. Teste: 
    choco -v

Se funcionar, agora você pode rodar:

choco install maven -y

Depois, confira o Maven:

mvn -v

3. Instalar Maven sem Chocolatey (alternativa)

  1. Acesse: https://maven.apache.org/download.cgi
    Baixe o Binary zip archive (por ex.: apache-maven-3.9.x-bin.zip).
  2. Extraia o ZIP, por exemplo para: 
    C:\dev\apache-maven-3.9.x
  3. Configure as variáveis de ambiente (Windows):
    • Abra Painel de Controle → Sistema → Configurações avançadas do sistema → Variáveis de Ambiente.
    • Em Variáveis do sistema:
      • Crie uma nova variável:
        Nome: MAVEN_HOME
        Valor: 
        C:\dev\apache-maven-3.9.x
      • Edite a variável Path e adicione: 
        C:\dev\apache-maven-3.9.x\bin
  4. Feche e abra novamente o PowerShell ou CMD e teste: 
    mvn -v

Se aparecer a versão do Maven e do Java, está instalado corretamente.
